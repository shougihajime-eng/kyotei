```
ユーザー仕様: 「世界一わかりやすく、世界一使いやすい」 必殺１ごうてい のアプリを作ってください。

# 発見した問題

ユーザー報告:
「PC をつけっぱなしにしていたのに、 一日に 1 件も推奨が出なかった」
「予想されない、 何のためのアプリか分からなくなる」

# 根本原因 (3 つ)

❶ ブラウザのタブが裏に回ると setInterval が止まる
   → 自動更新が動いていなかった
❷ 推奨 (buy/skip) ロジックの 「skip ゲート」 を積み重ねすぎて、 全レースが弾かれていた
   → 「精度を上げる」 ために条件を厳しくしすぎ、 結果として 「精度ゼロ件」 状態
❸ ユーザーが 「なぜ何も推奨されないのか」 を確認する手段がなかった

# 必殺１ごうてい でやってほしい修正 (順番に確認)

## 1. 自動更新の Page Visibility 対応

useEffect 内の setInterval は、 タブが非アクティブだと遅延・停止します。
visibilitychange イベントと focus イベントの両方を監視して、 タブが復帰した時に
最後の tick から所定の間隔以上経っていれば即 1 度更新するように直してください。

## 2. 推奨ロジックの 「skip ゲート」 を 20% 緩める

各スタイル (steady / balanced / aggressive 相当) の skip 閾値を一律約 20% 緩めて、
1 日 5-15 件は推奨が出る状態にしてください。 細かい数値は仕様によりますが、
原則として 「全部の関門を抜けるレースが 1 日 1 件以下」 になっているなら緩和必須。

## 3. 「最低 N 件保証」 セーフティネット

通常の推奨が N 件未満になった場合、 skip 判定の中から期待値が高い上位を救済して
「セーフティ買い (grade=C 相当の低信頼マーキング)」 として推奨に格上げしてください。
「世界一の精度」 より 「毎日 N 件以上の手数」 を優先する設計に。

## 4. 推奨内訳の見える化 (SkipBreakdownCard 相当)

ホーム画面に 「全 X 件中、 推奨 A 件、 見送り B 件、 待機 C 件」 を表示し、
主要な見送り理由 Top 3 もカウント。
推奨が 0 件 + 見送り 30 件超なら 🚨 赤バナー警告。

## 5. 「これを買え!」 巨大指示書 (BuyOrderHero 相当)

直前 (X-Y 分以内) の推奨 1 件を、 ホーム画面の最上部に超巨大表示:
- 🏆 勝負度: S/A/B/C
- ⚠️ 危険度: 低/中/高
- 🎯 買い目: N 点
- どこで (場名 / R 番号)
- 何を (本命買い目)
- いくら (金額)
- 💡 理由 (一言)
- 「📱 公式で今すぐ買う」 大ボタン
- 「初めての方へ」 注意書き
- 出現時にビープ音 (Web Audio API)

## 6. 「次の勝負レース」 「今日は見送り推奨」 (BattleModeCard 相当)

直近 (X-Y 分) の推奨が無くても、 中期 (Y-Z 分) の推奨を 「次の勝負レース」 として、
推奨が今日 0 件なら 「🛡️ 見送り推奨」 を派手に表示。 「常に何か出す」 設計。

## 7. 当たったレースを派手に (TodayHitsCard 相当)

「🎉 今日の的中 N 件 / 累計 +XX,XXX 円」 + 連勝マーク 🔥 + 各的中の詳細。

## 8. 買い忘れ警告 (MissedBuyCard 相当)

推奨が出ていたのに記録されなかったレースの結果を集計:
- 的中していたら 「+XXX 円逃した」 (機会損失警告)
- 外れていたら 「XXX 円損失回避 (見送り正解)」

## 9. ヘッダに件数バッジ常駐

どのタブにいても 「💰 N 件」 が見えるようにロゴ隣に常時バッジ。

## 10. 自動更新カウントダウン

「⏱ 次の自動更新まで N 分 N 秒」 を 1 秒刻みで常時表示。
「動いてない感」 を解消。

1〜10 を順番に実装してください。 1 つずつコミット & プッシュしてください。
時間がかかってもいいので妥協せず全部やり切ってください。
```
