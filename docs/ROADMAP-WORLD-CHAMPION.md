# 世界一を目指す 30 日ロードマップ

> 2026-05-07 起案 / Round 120 から始動

このドキュメントは「精度世界一の競艇予想アプリ」 を目指すための、**Round 120 〜 Round 130** の中期計画。
各 Round で何をやるか・なぜやるか・終わったらチェックを入れていく。

---

## 🎯 ゴール

- **業界トップクラスの ROI 110-120%** を狙える状態に持っていく
- 「絶対当たる」 は競艇の構造上不可能 (人が乗る船のレース、運の要素が残る)
- **「業界で一番賢く立ち回るアプリ」** を目標にする

## 📋 ロードマップ

### ✅ Round 119 までの状態 (起算点)
- 24 場の会場特性が完璧 (大村+6 / 戸田-7 など)
- モーター / 展示タイム / スタート / 風 / 波 / チルト / 部品交換 / 気配 / 進入予想 まで全部入り

### 🟢 Round 120: 取得範囲拡張 (データ蓄積の土台)

**問題**: 朝アプリを開いても、予想されるのは **発走±60分の最大 12 レースだけ**。
他のレース (午後・夕方) は出走表すら取得していない → 予想ゼロ。

**やること**:
- `refreshAll` の取得範囲を「今日の残り全レース」 に拡張
- API 並列度制限を実装 (boatrace.jp に過度な負荷をかけない)
- 直近 3 時間以内のレースは オッズ + 直前情報も取得、それ以外は出走表のみ
- 結果として朝起動 → オッズが出始めた瞬間から予想保存される

**期待効果**: 1 日あたり 100+ レースが予想保存される → 3〜5 日で学習機能が動き始める

---

### 🔵 Round 121: 記者予想 (公式◎○▲△) の取得・反映

**問題**: 現状はアプリ独自の AI 予想のみ。記者の目線が入っていない。

**やること**:
- boatrace.jp の公式予想 (◎○▲△ + 短評) を取得する API を追加
- 予想ロジックに 「記者印補正」 を統合 (◎は+5%、○は+3% など)
- 「記者の予想」 を画面でも見れるパネル追加

**期待効果**: 「機械×人間の集合知」 で精度 UP

---

### 🔵 Round 122: 選手のコース別実績

**問題**: 選手の総合勝率は使っているが、**「この選手が 1 コースで何 % 勝つか」** という情報は無い。

**やること**:
- boatrace.jp 選手ページから「コース別 1 着率 / 2 着率」 を取得
- 「3 コースから来た時の選手 X の本来の力」 を予想に反映

**期待効果**: 進入が想定外でも 「その選手のそのコースでの強さ」 で予想精度 UP

---

### 🔵 Round 123: ピット離れタイム

**問題**: 展示 ST は使っているが、**ピット離れタイム** (ピットアウトから第 1 マークまでの時間) は別データで未活用。

**やること**:
- boatrace.jp 直前情報からピット離れタイムを取得
- 「ピット離れが極端に遅い」 = 整備不良の可能性 として減点

**期待効果**: 「展示は良いがピット離れで崩れた」 を予測可能に

---

### 🟡 Round 124: 学習機能を 5 因子全部に拡張

**問題**: 今の学習は「1 号艇本命的中率」 1 因子のみで、 イン有利度の重みを ±0.02 動かすだけ。
モーター / 展示 / スタート の重みは固定。

**やること**:
- 5 因子それぞれの重み (inAdvantage, motor, exhibition, startPower, oddsValue) を学習対象に
- 過去成績から「モーターを重く見るべき日 / 軽く見るべき日」 を学習
- 過学習防止は維持 (3 期間安定確認 + 自動ロールバック)

**期待効果**: アプリが自動で学習・進化する

---

### 🟡 Round 125: 機械学習 (ロジスティック回帰) への置き換え

**問題**: 現状は線形スコア + softmax の単純モデル。 機械学習で精度をもう一段上げる。

**やること**:
- `lib/predict-ml.js` を新設し、ロジスティック回帰モデルを実装
- 過去の確定データを学習データに使用
- 既存ロジックと並列実行して、 ROI が安定して高い方を採用

**期待効果**: 5 因子の非線形相互作用を捉えられる → 精度 UP

---

### 🟡 Round 126: 選手の得意技プロファイル

**やること**:
- 過去の 1 着レースから「まくり屋」「差し屋」「イン逃げ屋」 を選手別に自動分類

### 🟡 Round 127: 節間調整 (前検日からの仕上がり推移)

## 🚫 やらないこと (思想)

- **「絶対当たる」 を約束しない** — 構造上不可能だから
- **無理な精度向上のためにデータの質を犠牲にしない** — 質の悪いデータで学習すると逆に下がる
- **過剰補正 / 過学習** — 各補正は ±5〜10% 以内、 3 期間安定確認

---

## 💡 思想

> 競艇は人が乗る船のレース。 運の要素は必ず残る。
> だから 「絶対当たる」 ではなく、 「業界で一番賢く立ち回る」 を目標にする。
> データを溜めて、 学習して、 階段状に精度を上げていく。 1 ヶ月で世界が変わる。
