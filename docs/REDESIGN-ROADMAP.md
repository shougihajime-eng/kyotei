# 万舟研究所 — リデザイン & 信頼性ロードマップ

> 起案日: **2026-05-10**
> 発端: shoug さんから「全レース並べてるだけ／真っ白で安っぽい／予想できているか分からない」 3 連打の指示
> 担当: Claude Code (kyotei-app セッションから読み書き)

このロードマップは、 既存の `ROADMAP-WORLD-CHAMPION.md` (= 予想精度向上) とは別軸の、 **「アプリの信頼性 + 体験 (デザイン/操作性) を本物の研究所水準に引き上げる」** 中期計画。

---

## 🎯 ゴール

- アプリを開いた瞬間に「今日はここが危ない」「ここ勝負」「ここ見送り」が直感で分かる
- PC でもスマホでも、 別物として最適化された体験
- 「予想できているのか分からない」 不安をゼロにする (動作の可視化)
- 毎日開きたくなる「研究所」 の世界観 (黒・濃紺ベース／赤・黄アクセント／妥協なし)

---

2. 「🔬研究所」 タブ (Stats / MansyuLab) を見ていた → ここも世界観が統一されていない

5. 予想自体が動いていない (= データ取得失敗で空っぽ → 何も無くて「真っ白」)

→ **Phase 0 で本番を実物確認して仕分ける**のが最優先。

## 🗺 Phase 一覧 (上から順に消化)

### Phase 0 — 本番実物確認 + 動作仕分け 【最優先・小】

**目的**: 「直すべき所」と「実は動いてる所」を分ける。 これをやらずに作り直すと無駄打ち。

**やること**:
1. 本番 https://kyotei-two.vercel.app/ を shoug さんが開く
2. 4 タブ (🏠ホーム / 📋一覧 / 🔬研究所 / ⚙️設定) を順番に確認
3. 各タブで以下をチェック (10 項目):
   - [ ] 更新エラーが出ていないか
   - [ ] 表示崩れがないか
   - [ ] ボタンが押せるか
   - [ ] データが取れているか (5 場のレースが見えるか)
   - [ ] PC とスマホでレスポンシブ崩れがないか
   - [ ] 無限読み込みになっていないか
   - [ ] 古い情報のまま固まっていないか
   - [ ] クリックして詳細が開くか
   - [ ] 予想 (荒れスコア・買い目・理由) が出ているか
   - [ ] 保存 (Supabase 同期) が動いているか
4. 問題のある箇所を「画面名 + 症状」 のリストにする → Phase 1 以降の対象が確定

**受け入れ基準**: 「真っ白／予想されてない」と感じる画面が、 どの画面のどの部分か特定できている。

---

### Phase 1 — 一覧タブ (RaceList) のリデザイン 【小〜中】

**目的**: 「全レース機械的に並んでいる」 印象を解消し、 5 場・荒れ前提のトーンに統一する。

**現状の問題 (推測)**:
- 「買うべきレース／オッズ確定待ち／オッズ取得不可／見送り」 のセクション分けが EV 時代の名残
- 白系トーン (`var(--bg-card)`) で MansyuTop の黒・濃紺と世界観がズレる
- 「👉 買い (N 点)」 「EV 1.20」 など EV 用語が残っている

**やること**:
- セクション名を万舟研究所トーンに変更 (「🚨 激荒れ警報レース」「⚠️ 荒れ注意」「⏳ 締切待ち (荒れスコア取得前)」「📭 見送り (荒れ条件足りず)」)
- カードを MansyuTop と同じ黒ベース＋赤・黄アクセントに統一
- EV 表記を「荒れスコア」 表記に置換 (または併記)
- 5 場以外のレースが万一混入していないか動作確認 (App.jsx 1038 のフィルタが効いているか)

**受け入れ基準**: 一覧タブを開いて MansyuTop と「同じアプリ感」 がする。 「全場全レース」 印象がなくなる。

---

### Phase 2 — スマホ専用最適化 【中】

**目的**: 片手操作・通勤中・寝床で「気持ちよく使える」状態にする。

**やること**:
- 主要操作ボタン (更新・タブ切替・詳しく見る) を **下 1/3 内**に集約 (親指届くゾーン)
- カード幅を画面幅 100% にし、 横スクロール完全排除
- 文字サイズ 14px 以上、 タップ領域 44px 以上を全体ルール化
- 上部固定ヘッダの高さを抑える (今の Header.jsx は高め)
- 折りたたみ詳細の開閉アニメをスムーズに
- iOS Safari の bounce / safe-area-inset 対応点検

**受け入れ基準**: iPhone で片手で「更新 → 一番上のカードを開く → 買い目を見る → 戻る」 が 5 秒以内・親指だけで完結する。

---

### Phase 3 — PC 専用最適化 【中】

**目的**: PC は情報密度を上げる。 スマホと同じ 1 列レイアウトは PC では「すかすか」。

**やること**:
- ホームを 2-3 カラムダッシュボード化 (左: 激荒れ警報 / 中央: 荒れ注意 / 右: 監視中サマリ)
- ホバー時に追加情報が出るインタラクション (スマホはタップ前提なので別)
- キーボード操作 (タブキーで移動・Enter で開く・Esc で閉じる) 対応
- 1920px ワイドモニタでも余白がスカスカにならない max-width 調整

**受け入れ基準**: PC で開いて「ダッシュボードを見ている」 感覚になる。 縦スクロールがほとんど不要。

---

### Phase 4 — デザイン世界観の刷新 【大】

**目的**: 「研究所」 「危険」 「万舟」 の世界観を、 装飾ではなく構造で表現する。

**やること**:
- カラーパレット定義 (今は inline style で散在 → CSS 変数化 / Tailwind config に集約)
- タイポグラフィ階層 (見出し / 本文 / 数値の役割を明確化、 数値専用フォント `tabular-nums` 統一)
- アイコンセット統一 (絵文字を活かしつつ、 重要操作はアイコンセットに置換検討)
- ローディング演出 (今の「🔄 更新中…」 は素っ気ない → 「研究所がデータを解析中」 風)
- 空状態の演出 (今の「🌙 今は荒れそうなレースなし」 は良いが他画面でも使いまわす)
- BOAT RACE 公式 / ボートピア系の整理感を参考にしつつ、 もっと現代的・ダーク・高級感

**受け入れ基準**: 「真っ白／安っぽい」 が消える。 アプリを起動した瞬間に「研究所だ」 と分かる。

---

### Phase 5 — 動作信頼性の総点検 【中・常設】

**目的**: 「予想できているか分からない」 不安をゼロにする。

**やること**:
- 取得 → スコア計算 → 判定 → 買い目 → 保存 の各段階に**進捗ログをユーザー側に可視化** (今もある程度あるが分かりにくい)
- 失敗時のエラー文言を専門用語抜きで整理 (例: 「データが取れませんでした (公式サイトが応答していません)」)
- 「予想 0 件」 のとき「本当に 0 件 (荒れ条件足りず)」 と「予想自体が動いていない (バグ)」 を区別表示
- 自動更新の状態を常時バナー表示 (「🟢 5 場・48 レース監視中・次回更新まで 2 分」)

**受け入れ基準**: shoug さんが「ちゃんと動いている」 と一目で確信できる。

### Phase 6 — 機能拡張 【大・任意】

CLAUDE.md の Phase 3 候補から、 デザイン/信頼性が落ち着いてから着手:
