## 違和感が出たときのデバッグ手順

### A. Header の同期ステータスが「⚠️ 同期失敗」 のまま

1. Settings 「☁️ クラウド同期」 → 「🔄 今すぐ全件同期」 で詳細エラー確認
2. ブラウザ DevTools → Network タブで Supabase へのリクエスト確認
3. Console タブでエラー詳細 (例: `RLS violation`, `429 rate limit`)

### B. PC と iPhone で件数が違う

1. PC で 「☁️ クラウド同期」 → 「🔄 今すぐ全件同期」 → stats: `pulled` / `pushed` の数を確認
2. iPhone でも同じ → cloud と local の数を比較
3. 差分があれば: GC で削除された 90 日超の AI 記録の可能性 (`📅 最古` を確認)

### C. ログイン後にデータが消えた!

1. **そのまま閉じない** (ブラウザを閉じるとリカバリ難しい)
2. ブラウザを開いたまま 「☁️ クラウド同期」 → 「🔄 今すぐ全件同期」
3. それでも消えていれば: Supabase ダッシュボード → Table Editor で `predictions` を確認
4. cloud にもなければ最後に書き込まれたタイミング以降のデータが失われた可能性 (Round 46 で対策済のはず)

### D. クラウドにデータがプライベートに残っているか

Supabase ダッシュボード → Authentication → Users で:
- `${username}@kyotei.local` (擬似 email) のユーザーが見える
- そのユーザーを Delete user → 関連 predictions も RLS により消去 (cascade)
