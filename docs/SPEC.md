## 0. アプリの一行説明

**「指定 5 場の今日の荒れレースだけを見つけて、 5,000 円でどう買うかを提示するアプリ」**

ユーザーは何も入力しない。 アプリ側がすべて決める。

### 0.1 思想 (両立すべき 2 つ)
- **初心者は超簡単に使える** (1 秒で勝負あり/なし、 3 秒で買い目)
- **上級者は深掘りできる** (折りたたみ + 研究所タブ)

「シンプル＝情報不足」 にしてはいけない。 ホーム画面で **買い・見送り・AI 評価・理由** まで即見える状態にする。 「なぜ買いなのか」 を 2 タップ以上奥に隠すのは**禁止**。

## 4. トップ画面 (ホーム) に出すもの

### 上部 (ヘッダー — 最小限)
- 🌊 万舟研究所 (ロゴ・黄色)
- 「🔄 今すぐ更新」 ボタン
- 今日の累計収支 (例: `+12,000円` / `−5,000円`)
- ログイン状態アイコン (Supabase 同期確認用 — 小さく)

### 中央 (今日の勝負)

#### 勝負レースが 1 件以上ある場合
レースカードを 1 件ずつ縦に並べる。 各カードには **以下 7 項目すべてが折りたたまずに即見える状態** で並ぶ:
1. **対象場** (戸田・江戸川…)
2. **レース番号** (◯R)
3. **締切までの時間** (例: 「あと 7 分」 ・5 分以内は赤点滅)
4. **判定** (🚨 激荒れ警報 / ⚠️ 荒れ注意 / 🔵 見送り推奨)
5. **荒れスコア / AI 評価** (◯◯/100 + 「強気」「弱気」 等の言葉)
6. **5,000 円の買い目** (例: 「3連単 1-2-4 / 1-2-3 各 2,500 円」)
7. **理由 3 行** (なぜ荒れるか・誰が来そうか・なぜこの買い目か)

⚠️ 重要: 6 と 7 を「詳しく見る」 ボタンの中に隠すのは**禁止**。 必ず最初から見える。

#### 勝負レースが 0 件の場合
- 大きく「**今日は勝負なし**」 (画面中央)
- サブテキスト: 「5 場で今日 N レース監視しましたが、 荒れ条件が揃ったレースはありませんでした」
- 何もしないことを肯定する文言 (「無理に買わない方が研究所として正しいです」)

### 下部 (各カードの中・折りたたみ)
タップで開閉する詳細セクション:
- スコア内訳 (進入不安/強風波/1号艇不安/攻め手/展示異変/オッズ妙味 6 軸)
- 気象・水面 (風速・波高・天候)
- 公式リンク (出走表 / オッズ / 直前情報 / 結果)
- レース終了後: 結果 (1 着 2 着 3 着 + 配当)
- レース終了後: 5,000 円買って当たったか・収支

## 5. やらないこと (絶対禁止)

| やらないこと | 理由 |
|---|---|
| 金額をユーザーに入力させる | 設定アプリにしないため |
| 予想スタイル 3 択 | 既に削除済み (Round 168) |
| 全 12 レース機械的表示 | 5 場・75 点+ のみ |
| 対象外場 (大村・住之江 等) の表示 | 5 場限定の徹底 |
| 「📡 監視中: N レース」 数字バッジ | 「全レース見てる」 誤解の元 |
| 「🤐 見送り N 件」 数字バッジ | ユーザー視点で意味不明 |
| 「😱 万舟見逃し N 件」 数字バッジ | 心理的ノイズ・ユーザー向けでない |
| 「💰 リアル/🧪 エア」 切替ボタン | 5,000 円固定なので不要 |
| 「💰 N 件 buy」 件数バッジ | 件数より「今日勝負あり/なし」 が大事 |
| 設定画面を最初に見せる | 開いた瞬間に勝負が分かるべき |
| 自動ログイン・自動購入 | 全プロジェクト禁止事項 |

## 6. タブ構成 (画面下固定 4 タブ)

| タブ | 名前 | 内容 | 入口の大きさ |
|---|---|---|---|
| 🏠 | ホーム | 上記 4 章 (厳選された勝負レースカード or「今日は勝負なし」)。 **これがメイン** | 大 |
| 📒 | 履歴 | 過去の判定・的中・収支のログ (シンプル一覧) | 中 |
| 🔬 | 研究所 | 上級者向け深掘り画面。 詳細は §6.1 参照。 **入口は小さく** (一般ユーザーを迷わせない) | 小 |
| ⚙️ | 設定 | 4 項目だけ (詳細は §6.2)。 大量の設定項目を置くのは禁止 | 小 |

旧「📋 一覧」 タブは **完全廃止** (= 全 12 レース機械的並べはダメ。 厳選はホームに統合)。

### 6.1 「🔬 研究所」 タブの中身

99% のユーザーは研究したくない。 研究所は**深掘りしたい人専用**。 「研究所をメインにしない」 こと。

中身:
- **AI 根拠** (なぜこの判定か・スコアの内訳・どの要因が効いたか)
- **学習データ** (見送り正答率・取りこぼし額・荒れスコア重み調整)
- **モーター傾向** (場別モーターの今節成績)
- **会場傾向** (5 場それぞれの今節の決まり手分布・配当傾向)
- **万舟分析** (過去の万舟出現パターン・期待値分析)

### 6.2 「⚙️ 設定」 タブの中身 (4 項目だけ)

| 項目 | 必須度 | 理由 |
|---|---|---|
| ログイン (Supabase) | 必須 | 端末間同期 |
| ログアウト | 必須 | アカウント切替・離脱 |
| データ削除 (リセット) | 必須 | 緊急時の全消去 |
| 通知 ON/OFF | 必須 | 「今ここだけ勝負」「期待値急上昇」「危険レース」 通知の土台 |

⚠️ 禁止: 資金・1 日予算・1 レース上限・スタイル選択・セーフティ買い ON/OFF など、 思想を崩す設定は**追加しない**。

## 8. 100 点の定義 (受け入れ基準)

このすべてが満たされて初めて「100 点」と認める:

- [ ] アプリを開いて **1 秒以内**に「今日勝負あり / なし」 が分かる
- [ ] 勝負ありの場合、 各レースの **対象場・R・締切・判定・スコア・買い目・理由 3 行** が **3 秒以内**に視認できる (折りたたみの中に隠さない)
- [ ] 理由は **3 行以内**で、 専門用語抜きで読める
- [ ] ユーザーは **何も入力しなくていい** (利用同意の初回チェック以外)
- [ ] **設定画面を 1 度も開かなくていい** (通知 ON/OFF だけ任意で開く)
- [ ] **金額・スタイル・場の選択肢が画面のどこにも無い**
- [ ] スマホで **片手・1 タップ**で各レースの詳細が開閉する
- [ ] PC でも **2-3 列のダッシュボード**として情報密度が上がる
- [ ] データ取得失敗時に **「⚠️ 公式サイトから取れませんでした」** の状態が明示される
- [ ] 黒・濃紺ベース、 赤・黄アクセントで「研究所感」 がある (色数 3 系統以下)
- [ ] **対象外場 (大村・住之江 等) のレースが画面のどこにも出ない**
- [ ] **シンプルだが情報不足ではない** — 「なぜ買いか」 「なぜ見送りか」 がホーム画面で読める
- [ ] **研究所タブの入口が小さい** (一般ユーザーが迷わない位置・サイズ)
- [ ] 上級者は研究所タブで **AI 根拠・モーター傾向・会場傾向・万舟分析** まで深掘りできる
